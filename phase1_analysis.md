# restaurant-cost-estimator フェーズ1改善案（2026-06-04）

## 分析概要
v1.2.0実装後の新規改善ポイント（P1-P4優先度付け）

---

## P1: 優先度最高（実装推奨）

### 1. 入力値ゼロ/負数時の視覚的フィードバック改善
**目的**: ユーザが計算失敗の理由を理解しやすくする
**現状**: 計算結果が非表示になるだけで、エラー理由が不明
**仕様**:
- 外食費が 0 円以下の場合：赤枠 + ⚠️ エラーメッセージ表示
- 月の外食回数が 0 の場合：同様にエラーメッセージ表示
- メッセージテンプレート: 「外食費と回数は1以上の数値を入力してください」
- メッセージ色: #ef4444（赤）、フォント: 13px、アニメーション: 0.2s の fadeIn
**実装タスク**:
- 入力フィールドに `onchange` イベントリスナーを追加
- エラーメッセージ用 div を結果エリアの上に挿入
- バリデーション関数 `validateInputs()` を新規作成
**DoD**:
- [ ] エラーメッセージが赤枠で表示される
- [ ] ユーザが値を修正して計算すると、エラーメッセージが消える
- [ ] モバイル・PC両環境で動作確認

---

### 2. パターン比較セクションの入力体験改善
**目的**: 複数パターン追加時のアクセシビリティ向上
**現状**: パターン追加後、スクロールしないと新規パターンフォームが見えない
**仕様**:
- 「+ パターンを追加」ボタン押下後、新規パターンフォームまでスクロールして自動フォーカス
- 新規パターンフォーム自体に `animation: slideDown 0.3s ease-out` アニメーション追加
- フォーカス後、パターン名フィールドに自動フォーカス（キーボード操作優先）
**実装タスク**:
- `addPattern()` 関数末尾に `setTimeout()` で 100ms 遅延後に `.scrollIntoView()` 実装
- 新規パターン div に CSS `@keyframes slideDown` を定義
- 新規パターンの最初の input に `autoFocus` 属性を追加
**DoD**:
- [ ] パターン追加ボタン押下後、自動スクロール確認
- [ ] 新規パターンがスライドダウンアニメーション付きで表示
- [ ] パターン名フィールドに自動フォーカス確認

---

### 3. 結果表示エリアの数値フォーマット改善
**目的**: 大きな数字の可読性向上
**現状**: `toLocaleString()` で 3 桁区切りはあるが、単位ラベル（「円」）の位置が曖昧
**仕様**:
- 月間費用・年間費用の下に「/月」「/年」という単位ラベルを明示的に追加
- フォント: 12px、色: #888、`margin-top: 2px`
- 金額テキストと単位を `flex` で横並びに配置（オプション）
**実装タスク**:
- HTML の `#output` と `#yearlyOutput` div の下に、単位ラベル span を追加
- 金額 div を親 div でラップして flex 配置に変更
**DoD**:
- [ ] 月間費用・年間費用の下に「/月」「/年」ラベルが表示される
- [ ] PC・モバイル両環境で見栄えが崩れない

## P2: 優先度高（推奨実装）

### 4. 複数パターン比較の削除確認ダイアログ
**目的**: 誤削除防止
**現状**: 削除ボタン押下で即座に削除（取り消し不可）
**仕様**:
- 削除ボタン押下時、`confirm()` ダイアログで「このパターンを削除してもよろしいですか？」と確認
- ユーザが「キャンセル」を選択した場合、削除を中止
- ユーザが「削除」を選択した場合、`removePattern()` を実行
**実装タスク**:
- `removePattern()` 関数の先頭に `if (!confirm(...)) return;` を追加
- HTML の削除ボタン `onclick` から直接削除処理を呼ばず、確認ダイアログを経由
**DoD**:
- [ ] 削除ボタン押下時にダイアログが表示される
- [ ] 「キャンセル」選択後、パターンが削除されない
- [ ] 「削除」選択後、パターンが削除される

---

### 5. モバイル表示時の結果グリッド改善
**目的**: 小画面での数値表示の可読性向上
**現状**: 480px 以下で月間・年間が 2 列並び（フォント 28px に縮小）
**仕様**:
- モバイル（480px 以下）では月間・年間を 1 列に積み重ねる（stack）
- 各欄の高さ・パディングを調整して、視認性を確保
- フォント: `clamp(20px, 5vw, 36px)` で responsive 対応
**実装タスク**:
- メディアクエリ 480px 以下で `grid-template-columns: 1fr;` に変更
- フォント size を clamp() で responsive に設定
**DoD**:
- [ ] iPhoneなどの小画面で月間・年間が縦並びになる
- [ ] テキストサイズが大きく見やすい

---

### 6. パターン比較グラフのスマートソート
**目的**: グラフの比較が容易になる
**現状**: パターンの追加順でグラフに表示（追加順と費用額が無関係）
**仕様**:
- グラフを表示する際、パターンを月間費用の降順（高い順）でソート表示
- グラフ内でのみソート（patterns 配列自体は変更しない）
- ラベルと金額の対応がズレないよう注意
**実装タスク**:
- `renderChart()` 内で `validPatterns.sort((a, b) => (b.price * b.times) - (a.price * a.times))` を実装
- ソート後のパターン順でバーを描画
**DoD**:
- [ ] グラフが高い順に並ぶ
- [ ] パターンの追加順が変わらない
