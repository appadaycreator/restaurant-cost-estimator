## P1: 優先度最高（実装推奨）

### 1. パターン比較グラフのスマートソート（前回P2-6）
**目的**: グラフ表示時、高額順にソートして比較を容易化
**現状**: グラフが追加順で表示される（金額順ではない）
**仕様**:
- グラフ表示時にパターンを月間費用の降順（高い順）でソート表示
- グラフ内のみソート（patterns配列自体は変更しない）
- ラベルと金額の対応がズレないよう注意
**実装タスク**:
- `renderChart()` 内で `validPatterns.sort((a, b) => (b.price * b.times) - (a.price * a.times))` を実装
**DoD**:
- [ ] グラフが高い順に並ぶ
- [ ] パターン追加順が変わらない
- [ ] 編集時、グラフが自動的に再ソートされる

---

### 2. 入力値バリデーション強化（val2の検証）
**目的**: val2（月の外食回数）が0の場合のエラーフィードバック
**現状**: val1（外食費）の検証あり。val2が0でも計算実行される（`times || 1` で1がデフォルト）
**仕様**:
- val1・val2の両方が1以上の数値であることを検証
- どちらかが0以下の場合、明確なエラーメッセージを表示
- エラーメッセージ: 「外食費と回数は1以上の数値を入力してください」
**実装タスク**:
- `validateInputs()` で val2 の チェック条件を追加
- 現在の条件: `if (!val1 || val1 <= 0)` → `if (!val1 || val1 <= 0 || !val2 || val2 <= 0)`
**DoD**:
- [ ] val1=1000, val2=0 の場合、エラーメッセージが表示される
- [ ] val1=0, val2=10 の場合、エラーメッセージが表示される
- [ ] 結果表示が非表示になる

---

### 3. パターン入力時のキーボード操作改善
**目的**: パターン複数追加時のキーボード入力体験向上
**現状**: 
- パターン削除ボタンのみが`onclick`で実装（キーボード操作に対応）
- 他の入力フィールドは`onchange`/`oninput`（キーボード操作後、フォーカス喪失時に反応）
**仕様**:
- パターン名・金額・回数の各フィールドで Enter キー押下 → 値を確定・次フィールドにフォーカス移動
- Tab キー標準動作を保持
**実装タスク**:
- 各パターン入力に `onkeydown="handlePatternKeydown(event, ${p.id}, field)"` を追加
- `handlePatternKeydown()` 関数を新規実装（Enter で次フォーカス移動、Shift+Enter で削除確認）
**DoD**:
- [ ] パターン名 Enter → 金額フィールドにフォーカス
- [ ] 金額 Enter → 回数フィールドにフォーカス
- [ ] Tab キーは通常通り動作

## P2: 優先度高（推奨実装）

### 4. 結果コピーボタンの追加
**目的**: ユーザが計算結果を簡単に共有・記録できるようにする
**現状**: 結果表示のみで、クリップボードへのコピー手段がない
**仕様**:
- 結果エリア（月間費用・年間費用）右側に「📋 コピー」ボタンを配置
- クリック時、「月間費用: XXX円, 年間費用: XXX円」のテキストをクリップボードにコピー
- コピー成功時、短時間（2秒）トーストメッセージ「コピーしました！」を表示
- コピー失敗時（ブラウザ非対応等）、ボタン無効化
**実装タスク**:
- `navigator.clipboard.writeText()` で実装
- トーストメッセージ用の一時的な div を動的生成
**DoD**:
- [ ] コピーボタンが表示される
- [ ] クリック時、クリップボードにコピーされる
- [ ] 成功メッセージが2秒間表示される

---

### 5. パターン入力フィールドの アクセシビリティ向上
**目的**: スクリーンリーダー対応、キーボード操作の明確化
**現状**: 
- ラベルと入力フィールドが `for/id` で紐づいていない
- パターン削除ボタンが aria-label を持たない
**仕様**:
- 各パターンの label に `for="pattern-${p.id}-name"` を追加
- 各 input に id を付与: `id="pattern-${p.id}-name"` 等
- 削除ボタンに `aria-label="パターン「${p.name}」を削除"` を追加
**実装タスク**:
- HTML テンプレートに id/for 属性を追加
- aria-label を動的に設定
**DoD**:
- [ ] スクリーンリーダーで各フィールドが正しく読み上げられる
- [ ] Tab キーでフォーカス順序が正しい

---

### 6. パターン名プレースホルダー例の拡充
**目的**: ユーザが何を入力すべきか直感的に理解できる
**現状**: パターン名の例が「ランチ」のみ
**仕様**:
- プレースホルダーを複数例で循環表示（または複数例を同時表示）
- 例: 「ランチ、ディナー、カフェ、飲み会など」
- スタイル: プレースホルダー色を少し濃くして視認性向上
**実装タスク**:
- プレースホルダーテキストを「ランチ・ディナー・カフェ等」に変更
- または、フォーカス時にツールチップで例を表示
**DoD**:
- [ ] パターン名フィールドに分かりやすい例が表示される

**DoD**:

### 8. グラフの金額ラベル オーバーフロー対応
**目的**: グラフ内の金額テキストが、バーからはみ出さない
**現状**: バー幅が小さい場合、金額テキストがバーをはみ出る
**仕様**:
- バー幅が小さい（例：100px以下）場合、金額を右側に配置
- バー幅が大きい場合は、現在通りバー内に配置
- レスポンシブに対応
**実装タスク**:
- `renderChart()` で金額テキストの配置を条件分岐
**DoD**:
- [ ] 小さいバーでも金額が見える
